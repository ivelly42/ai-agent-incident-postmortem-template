## What This Incident Postmortem Covers

| Postmortem Area | What The Buyer Needs |
| --- | --- |
| Incident Timeline | Reconstruct trigger, agent action, detection point, mitigation, and recovery |
| Scope Drift | Compare approved files, commands, services, and data against what actually happened |
| Evidence | Attach logs, commands, URLs, screenshots, release digests, and reviewer notes |
| Root Cause | Separate prompt failure, tool failure, missing test, access boundary, and human handoff gaps |
| Impact | Record customer, production, security, cost, and delivery impact in one ledger |
| Corrective Actions | Track owner, due date, verification command, and regression guard for each fix |
| Handoff | Preserve changed files, blockers, follow-up, and unresolved evidence gaps |
| Revenue Proof | Prevent order intent, views, stars, forks, or downloads from being counted as revenue |
